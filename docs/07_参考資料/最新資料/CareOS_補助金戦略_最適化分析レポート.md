# Universal Care OS — 補助金・助成金戦略 最適化分析レポート

**作成日**: 2026年3月23日（3月23日 制度検証に基づく全面改訂）
**分析対象**: サンライズDXプロジェクト全補助金資料
**分析軸**: JVの座組 / 申請主体 / ビジネススキーム
**検証方法**: 各制度の公式サイト・公募要領を個別に確認し、補助率・上限額・申請主体要件を精査

---

## エグゼクティブサマリー

本レポートは、Universal Care OS（CareOS）プロジェクトの補助金・助成金戦略を包括的に分析し、JVの座組・申請主体・ビジネススキームの3軸から最適化提案を行うものである。

現行戦略では初年度最大4,400万円の補助金ポテンシャルを試算していたが、制度検証の結果、補助率の過大評価と申請資格の制約により、**現実的なYear 1獲得見込みは2,200万〜3,400万円**と下方修正する。

**主要な発見事項（制度検証済み）**:
1. サンライズは**株式会社**であり（社会福祉法人ではない）、中小企業庁系補助金への申請主体になれる可能性がある。ただし従業員数・資本金による中小企業要件の充足確認が必要。JV法人を別途設立する場合も、サンライズ自体が直接申請する選択肢も検討すべき。**小規模事業者持続化補助金はJV株式会社の設立直後では採択が不確実**
2. **東京都DX推進助成金の補助率は基本1/2（50%）であり、旧版の「3/4〜4/5」は誤り**。3/4は賃金引上げ計画がある中小企業のみ、4/5は小規模事業者+賃金引上げor働き方改革枠。自己負担が旧試算の倍（750万→1,500万）になる可能性
3. 「JV法人が申請」と「導入先施設が申請」の二層構造は極めて有効。ただし、**障害福祉ICT導入モデル事業（東京都）はR2〜R5年度のモデル事業で終了の可能性**があり、R7年度は「デジタル機器導入促進支援事業」（福祉保健財団）に移行。金額・要件の再確認が必要
4. 2年目以降の大型補助金（SBIR・新事業進出）への橋渡しとなる「決算1期」の準備が現行計画では手薄。**SBIRのPhase 1は約1,500万円**であり「数千万〜数億」はPhase全体（最長3年）の話

## 第2部: 3軸分析

### 2.1 第1軸 — JVの座組の最適化

#### 現状の課題

**課題1: JV法人の法人形態が未確定**

現行資料では「JV法人」を申請主体としているが、JVの法人形態（株式会社・合同会社・一般社団法人等）が確定していない。法人形態によって申請可能な補助金が大きく異なる。

- 株式会社として設立 → 中小企業庁系補助金の大半に申請可能
- 一般社団法人として設立 → 一部の補助金で制限あり

> **重要**: サンライズは株式会社であるため、サンライズ自体が中小企業要件（サービス業: 資本金5,000万以下 or 従業員100人以下）を満たせば、サンライズ単体での申請も選択肢となる。JV法人設立が必須ではない補助金もある。

**推奨**: JV法人は**株式会社**として設立し、東京都に本店を置くこと（東京都DX推進助成金の要件充足のため）。

> **重要な発見**: サンライズの本店登記は**東京都多摩市**にある（事業所は横浜市青葉区）。これにより、**サンライズ単体でも東京都DX推進助成金に直接申請可能**であり、Track A（JV法人）だけでなくTrack B（サンライズ）からも東京都の補助金にアクセスできる。2トラック戦略の柔軟性が大幅に向上する。

**課題2: 3社の役割分担と知財・収益配分の明確化**

補助金申請においては、各社の役割分担と知財帰属が審査項目となる。現状は以下が不明確:
- CareOSの知財（ソースコード・アルゴリズム・データモデル）の帰属先
- 「わかるじゃんマトリックス」の知財はサンライズに帰属するが、デジタル化後の知財は？
- 補助金で開発した成果物の知財管理規約

**課題3: 「みなし大企業」規定への抵触リスク**

新事業進出補助金の公募要領によると、以下の場合は「みなし大企業」として補助対象外:
- 発行済株式の1/2以上を同一の大企業が所有
- 役員の過半数が同一の大企業の役員を兼務
- JV構成員の出資総額の過半数が大企業またはみなし大企業

DataBee・FDSiの資本構成を確認し、みなし大企業に該当しないことの検証が必要。
→ **検証完了**: 全社中小企業（資本金5,000万以下/従業員100人以下）、最大持株比率45%（<50%）。みなし大企業に該当しない。

**課題4: サンライズの公金収入による助成適格性制約**

サンライズの主たる収入は障害福祉サービス報酬（自立支援給付＝公定価格＝公金）であり、これが一部の助成金で受給制限となる可能性がある。

- **持続化補助金**: 障害福祉サービス給付費受給事業者は明示的に対象外（確定）
- **東京都振興公社の助成（DX推進・知財助成等）**: 株式会社で法人形態上は対象だが、公金収入事業者の詳細適格性は募集要項PDFに依存。⚠要確認
- **厚労省ICT導入支援事業**: 福祉施設向け制度のため、むしろ対象（◎適格）
- **特許庁中小企業減免**: 法人形態ベースの減免制度。公金収入による除外規定なし（◎適格）

→ **戦略方針: JV法人（CareOS株式会社）経由での申請を全助成金の主軸とする。** JV法人はSaaS開発会社であり公金収入の制約を受けない。サンライズ直接申請（Track B）はICT導入支援事業（厚労省）のみ確実。振興公社の助成はサンライズ直接でも可能性はあるが、事前に「障害福祉サービス報酬が主収入の株式会社」の適格性を確認すること。

#### 最適化提案: JV座組の推奨構成 — 段階的スキーム

**重要: 関連会社制約の回避**
補助金の公募要領では「資本関係がある事業者への外注費は補助対象外」と明記されている。DataBeeがJV株主のまま開発を受託すると、初期開発費2,400万円が全額対象外になるリスクがある。これを回避するため、段階的な資本参画スキームを採用する。

```
【Phase 1: JV設立〜補助事業完了】
┌─────────────────────────────────────────────┐
│  CareOS株式会社（JV法人・東京都所在）        │
│  ─────────────────────────────────────────── │
│  出資比率: サンライズ 75% / FDSi 25%          │
│  代表取締役: サンライズ側（福祉の信頼性）     │
│  ─────────────────────────────────────────── │
│  DataBeeは資本関係のない純粋な外注先          │
│  → JV→DataBee開発委託費 = 全額が補助対象経費  │
│  → JV→FDSi AI開発委託も補助対象経費           │
│  ─────────────────────────────────────────── │
│  補助金申請: JV法人が一元的に管理              │
│  知財: JV法人に帰属（各社にライセンスバック）  │
│  マトリックス原典: サンライズに永久帰属         │
└─────────────────────────────────────────────┘
              ↓ 補助事業完了後
【Phase 2: DataBee増資引受で資本参画】
┌─────────────────────────────────────────────┐
│  出資比率: サンライズ 45% / DataBee 40% / FDSi 15% │
│  → DataBeeがガバナンス・配当権を正式取得       │
│  → みなし大企業チェック全項目クリア             │
│    （最大45%<50%、全社中小企業）                │
└─────────────────────────────────────────────┘
```

**段階的スキームの根拠**:
- Phase 1でDataBeeを外注先に特化 → 開発委託費が全額補助対象経費
- Phase 2でDataBeeが増資引受 → ガバナンス・配当権を正式取得
- 両社長のコントロール下にあるため、資本参画タイミングで問題は発生しない
- みなし大企業チェック: Phase 1は2社（75/25）でリスクゼロ、Phase 2は3社（45/40/15）で全項目クリア

**各社の補助金申請における最適パートナー**:

| 補助金 | 最適申請主体 | パートナーの役割 |
|--------|-------------|-----------------|
| DX推進助成金 | JV法人（CareOS株式会社） | DataBee=開発委託先, FDSi=AI委託先 |
| 創業助成事業 | JV法人 | サンライズ=実証フィールド提供 |
| 障害福祉ICT導入支援 | サンライズ（各施設） | JV法人=システム提供者 |
| 新事業進出補助金 | JV法人 | 3社連携体での申請も検討 |
| Go-Tech事業 | JV法人 | 大学・研究機関との連携が加点要素 |
| SBIR | JV法人 | 厚労省テーマ（虐待予防AI）で申請 |

### 2.2 第2軸 — 申請主体の最適化

#### 申請主体ごとの採択確度分析

**パターンA: JV法人（CareOS株式会社）が申請主体**

| 制度 | 採択確度 | 加点要素 | リスク |
|------|---------|---------|--------|
| 東京都DX推進助成金 | ★★★★ 高 | DXアドバイザー経由、福祉DXの社会性 | アドバイザー支援に3ヶ月必要 |
| 東京都創業助成事業 | ★★★★ 高 | 特定創業支援等事業の証明書、社会課題解決 | 創業5年未満の要件 |
| 小規模事業者持続化 | ★★★ 中 | 事業計画の具体性 | 上限250万で投資対効果が低い |
| TCiN | ★★★★ 高 | 藤井議員推薦、社会課題解決型 | 採択数が限定的 |
| 新事業進出補助金 | ★★★ 中 | 大型補助だが決算1期必要 | 創業1年未満は対象外 |

**パターンB: サンライズ（株式会社・障害福祉事業者）が申請主体**

> **重要**: サンライズは株式会社であり、中小企業要件を満たせば中小企業庁系補助金にも直接申請可能。
> 施設申請型の補助金に加え、サンライズ自体がIT導入補助金等に申請する選択肢もある。

| 制度 | 採択確度 | 加点要素 | リスク |
|------|---------|---------|--------|
| 障害福祉ICT導入支援 ⚠️ | ★★★★ 高 | 障害福祉事業所の実績、現場ニーズの明確さ | **東京都モデル事業はR5終了の可能性。R7「デジタル機器導入促進支援事業」に移行要確認** |
| デジタル化・AI導入補助金 | ★★★★ 高 | 業務効率化の定量効果 | IT導入補助金との併用不可 |
| 省力化投資補助金（サンライズ直接） | ★★★ 中 | 中小企業要件を満たせば申請可 | 従業員数・資本金の確認が必要 |
| 小規模事業者持続化補助金 | ★★★ 中 | サンライズが小規模事業者要件を満たす場合 | 従業員5人以下（サービス業）の要件 |

**パターンC: コンソーシアム型（連携体）で申請**

新事業進出補助金では「連携体」での申請が可能。JV法人（代表申請者）+ サンライズ（連携体構成員）+ 大学/研究機関で申請する方式。

| メリット | デメリット |
|----------|-----------|
| 補助上限額の引き上げ | 全構成員が要件を満たす必要 |
| 各社の強みを審査で訴求可能 | 事務手続きの複雑化 |
| 大学連携で技術的信頼性向上 | 利益相反管理が必要 |

#### 最適化提案: 申請主体の使い分け戦略

**フェーズ1（JV設立〜6ヶ月）: 即時申請可能な制度から着手**

1. **JV法人** → 東京都創業助成事業（400万）: 設立直後に申請。特定創業支援等事業の証明書を事前取得
2. **JV法人** → DXアドバイザー支援開始: 助成金申請の前段階。3ヶ月後にDX推進助成金へ
3. **サンライズ施設** → 障害福祉ICT導入支援（1,000万/施設）: 東京都福祉部局に事前相談（藤井議員経由）

**フェーズ2（6ヶ月〜12ヶ月）: 主力補助金の申請**

4. **JV法人** → 東京都DX推進助成金（3,000万）: アドバイザー提案書に基づく申請
5. **JV法人** → TCiN推薦: 藤井議員経由でマッチング支援

**フェーズ3（12ヶ月〜24ヶ月）: 大型補助金への展開**

6. **JV法人** → SBIR（Phase 1: 約1,500万、最長3年で最大9,750万）: 厚労省「障害者自立支援機器開発」テーマで研究開発計画策定
7. **JV法人（連携体）** → 新事業進出補助金（最大9,000万）: 決算1期完了後。大学との共同申請

### 2.3 第3軸 — ビジネススキームの最適化

#### 補助金ごとの事業スキーム設計

**スキーム1: 東京都DX推進助成金（上限3,000万円）**

> **⚠️ 補助率に関する重要な修正（2026-03-23検証済み）**
> - 中小企業の基本補助率: **1/2（50%）** ← 旧版の「3/4〜4/5」は誤り
> - 3/4: 賃金引上げ計画がある中小企業のみ
> - 4/5: 小規模事業者 + 賃金引上げ or 働き方改革枠
> - DXアドバイザー支援（無料・必須）を経由しなければ申請不可

```
[JV法人（CareOS株式会社）]
  │
  ├─ パターンA: 基本補助率1/2の場合
  │   補助対象経費6,000万 → 補助3,000万 / 自己負担3,000万
  │   ※自己負担が大きいため、対象経費を絞って申請する方が現実的
  │
  ├─ パターンB: 賃金引上げ計画で3/4に引上げの場合
  │   補助対象経費4,000万 → 補助3,000万 / 自己負担1,000万
  │   ※JV法人で従業員を雇用し、賃上げ計画を策定する必要あり
  │
  ├─ 現実的な申請プラン（補助率1/2、対象経費4,000万）
  │   ├─ SaaS開発費: DataBeeへの委託 → 2,000万
  │   ├─ AI開発費: FDSiへの委託 → 800万
  │   ├─ クラウドサーバー費 → 500万
  │   └─ その他（専門家費・広告等） → 700万
  │   → 補助額: 2,000万 / 自己負担: 2,000万
  │
  └─ 自己負担: 1,000万〜2,000万（DataBee開発投資から充当）
```

**補助率最大化のポイント**:
- **賃金引上げ計画の策定が最重要**（1/2→3/4への引上げで自己負担が半減）
- JV法人で最低1名を雇用し、賃上げ計画を立てることで3/4を狙う
- DXアドバイザーの提案書に「AI活用による福祉支援品質向上」を明記

**スキーム2: 障害福祉ICT導入支援（1,000万円/施設・補助率3/4）**

```
[サンライズ施設（事業所単位で申請）]
  │
  ├─ 補助対象経費（1,333万のうち1,000万が補助）
```
